How to Analyze COVID-19 Data¶

In this project, we will walk you through a tutorial of the data science pipeline as we explore trends in COVID-19 data. My name is Rejas Raj.

COVID-19 has wreaked havoc on nearly all countries in the world. Although the death rate of the disease is relatively low, millions of lives have been lost, so it's essential that we understand trends in the spread of the disease so that we can figure out where to focus our efforts to mitigate it. There's an abundance of COVID-19 data available on the internet, and there's a variety of ways we can analyze the data to gain valuable insights from it. Much of the analyses we will do in this project will focus on COVID-19 cases, deaths, death rates, and how they vary among different regions, because knowing this could potentially give us insight into whether COVID-19 behaves differently in different climates, for example.

Let's start by collecting data.

Data Collection¶

The first step of the data science pipeline is Data Collection, which is the process of retrieving data from the web and storing it into a data structure we can later manipulate and analyze. The COVID-19 dataset we will analyze in this project is the from the Center for Systems Science and Engineering (CSSE) at Johns Hopkins University. There are two tables we will retrieve: one consisting of data about COVID-19 cases, and the other consisting of data about COVID-19 deaths. Let's extract the data.

In [256]:
# We need the Pandas library to read the data from the web into a DataFrame.
import pandas as pd
# The Pandas read_csv function extracts data from a CSV (comma-separated values) file and stores the result into a DataFrame.
# Url for COVID-19 cases data

# Url for COVID-19 deaths data
url_deaths = "https://raw.githubusercontent.com/CSSEGISandData/COVID-19/master/csse_covid_19_data/csse_covid_19_time_series/time_series_covid19_deaths_global.csv"
# Read the COVID-19 deaths table into a DataFrame 
df_deaths = pd.read_csv(url_deaths)

We now have our two tables stored into a Pandas DataFrame. A DataFrame is a 2-dimensional data structure that stores tabular data. Fortunately, the raw data came from CSV (comma-separated values) files, and the Pandas library has a function that allows us to easily store a CSV file into a DataFrame. If the raw data was not stored in a friendly format that we can easily store in a DataFrame, we would need some way to parse the data. Parsing is a complex step of the data collection process that involves matching the format of the raw data to rows in a DataFrame, which could be done with regular expressions, for example. Fortunately, we were able to obtain our DataFrames without the need for parsing. Let's display each of these DataFrames.

Data Management¶

The Data Management step is where we organize the data we collected in a way that makes it ready to be analyzed. This often involves making our data tidy. Tidy data is a concept in data science that refers to a dataset that satisfies the following three properties:

  1. Every column is a variable.

  2. Every row is an observation.

  3. Every cell is a single value.

For more information about tidy data, check out this article: https://cran.r-project.org/web/packages/tidyr/vignettes/tidy-data.html. Unfortunately, our DataFrames don't satisfy all three properties of tidy data. The second property is violated because we have multiple observations for each row (one for each date). To tidy our data, we need to convert the date columns into rows. Specifically, for the cases DataFrame, we will let the date column headers be values of a variable called Date, and we will name the case count variable Cases. We will do something similar for the deaths DataFrame. This process is called melting, and it's performed below. To learn more about melting, check the documentation for the Pandas melt function: https://pandas.pydata.org/docs/reference/api/pandas.melt.html.

In [259]:
# First melt the cases DataFrame
# List of identifier columns
df_cases_id_vars = ["Province/State", "Country/Region", "Lat", "Long"]
# Name of variable who values are the names of all columns not in id_vars
df_cases_var_name = "Date"
# Name of observed variable
df_cases_value_name = "Cases"
# Melt the cases DataFrame
df_cases = df_cases.melt(
    id_vars = df_cases_id_vars,
    var_name = df_cases_var_name,
    value_name = df_cases_value_name
)
# Do the same for the deaths DataFrame
# List of identifier columns
df_deaths_id_vars = ["Province/State", "Country/Region", "Lat", "Long"]
# Name of variable who values are the names of all columns not in id_vars
df_deaths_var_name = "Date"
# Name of observed variable
df_deaths_value_name = "Deaths"
# Melt the deaths DataFrame
df_deaths = df_deaths.melt(
    id_vars = df_deaths_id_vars,
    var_name = df_deaths_var_name,
    value_name = df_deaths_value_name
)

Exploratory Data Analysis¶

The Exploratory Data Analysis step of the data science pipeline is where we create plots and graphs so that we can visualize our data to discover patterns or trends in it. As we do this, we will ask questions we are curious about that we may be able to answer in more depth later, in the Hypothesis Testing step. We will first create a line plot showing the total COVID-19 cases over time.

# We need the numpy library to help create the logarithmic scale
import numpy as np
# Create a figure and a set of subplots

# Compute the minimum cases for a country
min_cases = cases_for_each_country.min()
# Compute the maximum cases for a country
max_cases = cases_for_each_country.max()
# Compute the base-10 logarithm of the minimum cases, or set it to zero if the minimum cases is zero
if min_cases > 0:

The histograms above are the same as the previous ones except the x-scale is now logarithmic. This means that we can now see how the magnitude of the number of COVID-19 cases varies between countries. Since the distribution of COVID-19 cases seems to be centered between 10^5 and 10^6, we can say that a country is expected to have hundreds of thousands of COVID-19 cases. Similarly, since the distribution of COVID-19 deaths seems to be centered between 10^3 and 10^4, we can say that a country is expected to have thousands of COVID-19 deaths. In terms of spread, both distributions seem slightly left-skewed. This means that while most countries have a relatively high magnitude of COVID-19 cases and deaths, some have a much smaller magnitude of cases and deaths.

There are some tradeoffs involved in choosing a logarithmic scale over a linear scale. A benefit of a linear scale is that it shows the distributions of cases and deaths more directly, while the logarithmic scale indirectly shows the distributions of cases and deaths by showing how they vary in magnitude across countries. However, since we could not learn much from the plots with a linear scale since nearly all countries fell into the first bin of the histograms, a logarithmic scale is the better alternative.

We've done a lot of analysis of COVID-19 cases and deaths, but let's take a look back at the death rate. We analyzed the overall death rate over time, but let's take this further. What if we wanted to know whether the COVID-19 death rate is different in tropical climates than in non-tropical climates? Let's split the countries into two categories: countries in the tropics and countries outside the tropics. We'll produce a line plot showing how the death rate in tropical countries changed over time and how the death rate in non-tropical countries changed over time. The tropics are defined as areas on earth within 23.43638 degrees north or south of the equator. In other words, the tropics consist of regions between -23.43638 degrees longitude and 23.43638 degrees longitude. Fortunately, our DataFrame provides latitude and longitude information for each region, so we can filter out tropical regions and non-tropical regions using conditional statements involving the longitude information. This is done below.

In [269]:
# Create a figure and a set of subplots
fig, ax = plt.subplots()
# Set an appropriate title for the plot
plt.title("COVID-19 Death Rate Over Time for Tropical and Non-Tropical Regions")
# Get the slice of the DataFrame consisting of tropical regions
tropics = df[(df['Long'] >= -23.43638) & (df['Long'] <= 23.43638)]
# Get the slice of the DataFrame consisting of non-tropical regions
non_tropics = df[(df['Long'] < -23.43638) | (df['Long'] > 23.43638)]
# Group the tropics slice by date
tropics_group_by_date = tropics.groupby("Date")
# For each date, find the sum of the numeric columns for that date
tropics_group_by_date_sum = tropics_group_by_date.sum()
# Get the total cases by each date for the tropics slice
tropics_total_cases_by_date = tropics_group_by_date_sum["Cases"]
# Get the total deaths by each date for the tropics slice
tropics_total_deaths_by_date = tropics_group_by_date_sum["Deaths"]
# Group the non-tropics slice by date
non_tropics_group_by_date = non_tropics.groupby("Date")
# For each date, find the sum of the numeric columns for that date
non_tropics_group_by_date_sum = non_tropics_group_by_date.sum()

Hypothesis Testing¶

Let's come up with a way to test whether there is a significant difference in the current death rate between tropical and non-tropical regions. If we treat the death rates for the tropical regions and the death rates for the non-tropical regions as two independent samples, we can perform a two-sample t-test comparing the mean death rate between tropical and non-tropical regions. Let's do that below.

# We will need the scipy.stats library to perform a two-sample t-test
from scipy.stats import ttest_ind
# Add a column for death rate, estimated as deaths divided by cases
df["Death Rate"] = df["Deaths"] / df["Cases"]
# Get the slice of the DataFrame consisting of current data for tropical regions where the number of cases is nonzero (to avoid dividing by zero)
tropics = df[(df["Date"] == today) & (df["Cases"] != 0) & (df['Long'] >= -23.43638) & (df['Long'] <= 23.43638)]
# Get the slice of the DataFrame consisting of current data for non-tropical regions where the number of cases is nonzero (to avoid dividing by zero)
non_tropics = df[(df["Date"] == today) & (df["Cases"] != 0) & ((df['Long'] < -23.43638) | (df['Long'] > 23.43638))]
# Run a two-sample t-test comparing the mean death rate in the tropics to the mean death rate in the non-tropics
result = ttest_ind(tropics['Death Rate'], non_tropics['Death Rate'])
# Get the test-statistic of the two-sample t-test
test_statistic = result.statistic
# Get the p-value of the two-sample t-test
p_value = result.pvalue
# Print the test-statistic
print("Test-statistic:", test_statistic)
# Print the p-value
print("p-value:", p_value)

Above we have run a two-sample t-test to compare the mean death rate in tropical regions to the mean death rate in non-tropical regions. We will assume the death rates in tropical regions (as well as the death rates in non-tropical regions) follow a normal distribution. Our null hypothesis is that the mean COVID-19 death rate in tropical regions is the same as the mean COVID-19 death rate in non-tropical regions, and our alternative hypothesis is that the mean COVID-19 death rate in tropical regions is different from the mean COVID-19 death rate in non-tropical regions. We will test the null hypothesis at significance level 0.05. The test-statistic of the two-sample t-test, computed above, is -0.591, and its corresponding p-value is 0.555. Since p > 0.05, we fail to reject the null hypothesis. There's not enough evidence to conclude that the mean COVID-19 death rate differs between tropical regions and non-tropical regions.

Note that in choosing our hypothesis testing procedure, there are other alternatives we could have considered. For example, instead of comparing the mean of the death rates in tropical regions to the mean of the death rates in non-tropical regions, we could have compared the overall death rate in tropical regions with the overall death rate in non-tropical regions, where the overall death rate is total deaths divided by total cases. A benefit of this approach is that it more directly captures the death rates for these two different categories of regions, but it allows regions with higher cases and deaths to have more influence on the overall death rate than other regions. Our approach, on the other hand, ensures that each region has equal representation in the death rate, so our approach was chosen over the other approach.

Since we have tested our hypothesis, we have now completed the Hypothesis Testing step. Note that it isn't necessary to get a statistically significant result during hypothesis testing. All that matters is that we know what the results of the hypothesis testing step mean in the context of our data.
